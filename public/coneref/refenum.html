enums

Note: None of this capability is currently implemented.

enum is used to declare a collection of symbols that are mapped to integer values.

enum PhoneType:
	Home,
	Work,
	Mobile

These symbols are automatically mapped to unsigned integer values: Home to 0, Work to 1, and Mobile to 2. Use of a name is qualified by its enum name: PhoneType::mobile.

It is possible to explicitly specify the numbers the symbols map to: 

enum Colors:
	Red   = 0xFF0000,
	Green = 0x00FF00,
	Blue  = 0x0000FF

Even though these symbols map to integer values they are not aliases of one of the defined integer types (e.g., u8). They are a distinct type. Although numeric, they have no pre-defined arithmetic methods attached to them, so you cannot add, subtract, multiply or divide enumerated values.

Initialization

An enum initial value is just its qualified name:

phone.phonetype = PhoneType::Mobile

Comparing

An enum value may only be compared for equivalence (and not order).
